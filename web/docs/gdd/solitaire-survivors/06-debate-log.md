# Debate Log - 기획 토론 기록

## 기획 주제

"뱀파이어 서바이벌 + 솔리테어" 퓨전 게임 모드
**키워드**: `solitaire-survivors`

### 사용자 제약 조건
1. 상단 게임뷰, 하단 솔리테어
2. 플레이어 캐릭터 제자리 고정, 적만 지속 스폰
3. 특정 Foundation 조합으로 투사체 개수 증가

---

## Round 1: 독립 제안

### Classic Rule Agent 초안

#### 핵심 루프
- 실시간 동시 진행 (턴제 페이즈 분리 폐기)
- Klondike 규칙 100% 보존
- Foundation 적재 = 즉시 투사체 강화

#### 솔리테어→전투 연결 (3채널)
- Foundation 적재 → 투사체 수/속성 변화 (핵심)
- Tableau face-down 뒤집기 → 소량 경험치
- Stock 순환 → 0.5초 전체 적 감속

#### Foundation → 투사체 시스템
- Suit → 투사체 타입: Spade=검기(부채꼴), Heart=성탄오브(공전), Diamond=화살(유도), Club=화염구(범위)
- 투사체 수: `ceil(Foundation 최고 Rank / 3)`, 최소 1 최대 5
- Rank 구간 데미지 배율: A-4(x1.0), 5-9(x1.5), 10-K(x2.2)
- Suit 콤보: 같은 Suit 연속 적재 시 데미지/발사속도 보너스
- Full Suit(K) 완성: 투사체 특수효과

#### 레벨업 선택지 (VS 메카닉)
- 경험치: 적 처치 + Tableau 뒤집기
- 3택: 투사체 강화 / 생존 강화 / **솔리테어 보조**

#### 웨이브 구조
- 10분, 10 웨이브
- Suit 내성 웨이브 (Wave 5, 8)
- 최종 보스: King of Ruin

#### 리스크 (자가 진단)
1. 멀티태스킹 부담 (상단 전투 + 하단 솔리테어)
2. 솔리테어 막힌 상태에서 무기력감
3. 세션 길이와 솔리테어 소모 속도 불일치

### Creative Rule Agent 초안

#### 핵심 루프
- 실시간 동시 진행
- **양방향 피드백**: 솔리테어→전투 + 전투→솔리테어

#### 솔리테어→전투 (순방향, 5채널)
- Foundation 적재 → 즉시 투사체 1발 + 영구 Tier 상승
- Tableau→Tableau 이동 → "체인 게이지" 충전 (풀 시 전체 화면 공격)
- 카드 뒤집기 → 0.3초 무적 + 넉백 파동
- Stock 순환 → Shield Orb 생성 (최대 3개)
- 빈 Tableau 열 → "결계 영역" (적 감속 존)

#### 전투→솔리테어 (역방향)
- 적 처치 → "야생 카드" 드랍 (Joker/Mirror/Bomb)
- 보스 등장 → Tableau 열 "저주" (3초 조작 불가)
- 캐릭터 피격 → Foundation 적재 0.5초 불가
- 웨이브 클리어 → face-down 1장 자동 reveal

#### Foundation → 투사체 시스템
- Suit Tier (0~MAX): Foundation 높이에 따라 자동 투사체 0~5발
- **크로스 Suit 콤보** (6+ 조합): 2개 이상 Suit Tier 조건 충족 시 "융합 투사체"
- Rank 보너스: 높은 카드 적재 시 즉발 위력 증가 + K 궁극기

#### 웨이브 구조
- 12분 + Endless, 7 웨이브
- 솔리테어 막힘 안전장치: 5초 후 Tableau 셔플 (HP 15% 패널티)

#### 메타 진행도
- 런 간 영속 해금, 해금 캐릭터 3종

#### 리스크 (자가 진단)
1. 인지 과부하 (양쪽 모두 복잡)
2. 솔리테어 막힘의 좌절감
3. 밸런스 조율의 복잡성

## Round 2: 교차 비판 + 수정

### Classic의 Creative 비판

#### 인정할 점
1. 양방향 피드백 구조 — "전투→솔리테어" 역방향이 두 시스템의 단절감을 해소
2. 크로스 Suit 콤보 — "어떤 Suit를 먼저 올릴까" 중기 전략 목표 부여
3. 야생 카드 — 솔리테어 막힘의 창의적 완화책

#### 문제점
1. **인지 부하 임계 초과**: 20개 이상의 동시 추적 요소. 분석 마비(analysis paralysis) 유발
2. **부정적 역방향은 하향 나선 유발**: 보스 저주, 피격 패널티가 "약해질수록 더 약해지는" 구조
3. **구현/밸런싱 비용 과도**: 크로스 Suit 11조합 + 야생카드 3종 + 역방향 5종 + 메타 진행도

#### 대안
1. "액티브/패시브 분리 원칙" — 의식적 추적 요소 최대 2개로 제한
2. 부정적 역방향 전부 제거, 긍정적 역방향만 채택
3. 크로스 Suit → "듀얼 콤보" 3종(Red/Black/Royal)으로 축소

#### Classic 수정안 (v2)
- Creative의 양방향 피드백 수용 (긍정적만)
- 듀얼 콤보 3종 추가
- 야생 카드 2종(Joker + Bomb) 채택
- 결계 존(빈 열 = 감속) 채택
- 10분 10웨이브 유지, 메타 진행도는 스코프 외

---

### Creative의 Classic 비판

#### 인정할 점
1. Foundation→투사체 매핑의 명료함 — 수치까지 공식화되어 구현 모호함 없음
2. 레벨업 선택지 도입 — VS 핵심 재미 + "솔리테어 보조" 카테고리의 교차점
3. 막힘 대응의 실용성 — 경험치 대안 경로

#### 문제점
1. **단방향 연결**: 전투→솔리테어 채널이 레벨업 "솔리테어 보조"뿐. 긴장감 축 1개
2. **Rank 구간 배율의 역설**: A부터 순서대로 올려야 하므로 x1.0→x1.5→x2.2는 자동 진행, 플레이어 선택 아님
3. **Suit 내성 웨이브의 설계 모순**: Klondike에서 특정 Suit 우선 적재 전략이 거의 불가능 → 운에 의한 벽

#### 대안
1. 전술 토큰 도입 (적 처치 → Reveal/Swap/Wild 토큰)
2. Rank 배율 대신 "적재 속도 보너스" (콤보 타이머)
3. 고정 Suit 내성 → 개별 적 약점 표시 + 보스만 내성

#### Creative 수정안 (R2)
- Classic의 투사체 수 공식 수용 (ceil 공식)
- Classic의 레벨업 시스템 수용
- 콤보 타이머 도입 (8초 내 연속 적재 시 배율 증가)
- 개별 적 약점 시스템 도입
- 12분 10웨이브, 메타 진행도 유지

### 미합의 항목 판정

#### 항목 1: 데미지 스케일링 — Rank 구간 배율 vs 콤보 타이머

- **Classic**: Rank 구간별 고정 배율 (A-4 x1.0, 5-9 x1.5, 10-K x2.2). 직관적이고 단순.
- **Creative**: 콤보 타이머 (10초 내 연속 적재 시 배율 증가). 실력 기반 보상.

**Orchestrator 판정**: Creative 채택, 조건부.
**근거**: Classic의 Rank 배율은 Creative가 정확히 지적한 대로 "선택"이 아닌 "시간에 따른 자동 증가"이다. Foundation에는 A부터 순서대로만 올리므로 플레이어가 배율을 제어할 수 없다. 콤보 타이머는 "빠르고 정확한 연속 적재"라는 실력 기반 보상을 만든다. 다만 Classic의 우려(복잡성)를 반영하여: 콤보 타이머 윈도우를 8초→10초로 완화, Rank 소폭 보너스(A-9 x1.0, 10-K x1.3)를 보조로 유지.

#### 항목 2: Suit 내성 — 고정 웨이브 vs 개별 적 약점

- **Classic**: Wave 5, 8에 고정 Suit 내성 웨이브 배치.
- **Creative**: 개별 적에 Suit 약점 아이콘 표시 + 보스만 내성.

**Orchestrator 판정**: Creative 채택.
**근거**: Klondike 규칙 보존 전제 하에서 특정 Suit를 우선 적재하는 전략은 거의 불가능하다. 고정 내성 웨이브는 "운에 의한 벽"이 되어 플레이어 좌절을 유발한다. 개별 약점 표시는 정보를 주고 보상하되 처벌하지 않는다.

#### 항목 3: 크로스 Suit 콤보 범위

- **Classic v2**: 듀얼 콤보 3종 (Red/Black/Royal). 색상 기반, 직관적.
- **Creative R1**: 6+ 조합 (2-Suit 4종 + 4-Suit 2종). 다양하나 밸런싱 변수 폭발.

**Orchestrator 판정**: Classic 채택.
**근거**: 3종(Red/Black/Royal)은 색상 기반으로 직관적이고 밸런싱 변수 3개로 관리 가능하다. 6+ 조합은 이 프로젝트 규모에서 현실적으로 완성·밸런싱 불가능하다.

#### 항목 4: 전투→솔리테어 역방향 — 야생카드 2종 vs 전술토큰 3종

- **Classic v2**: Joker(7%) + Bomb(3%) = 2종. 간결, 각각 명확한 용도.
- **Creative R2**: Reveal + Swap + Wild = 3종. 다양하나 인지 부하.

**Orchestrator 판정**: Classic 기반, Creative에서 1종 추가.
**근거**: Classic의 Joker + Bomb은 간결하면서 듀얼 유즈(Bomb: Foundation=폭발, Tableau=뒤집기)가 전략적이다. Creative의 Reveal(미리보기)은 솔리테어 막힘 해소에 실질적이므로 추가. Swap은 Klondike 규칙 변형이라 제외. 최종: **Joker(7%) + Peek(5%) + Bomb(3%) = 3종**.

#### 항목 5: 세션 길이

- **Classic**: 10분.
- **Creative**: 12분.

**Orchestrator 판정**: Classic 채택 (10분).
**근거**: 솔리테어 기반 게임은 짧은 세션이 적합하다. 10분 내에 압축된 경험을 제공하되, Endless 모드는 확장으로 예약.

#### 항목 6: 메타 진행도

- **Classic v2**: v1 스코프 외.
- **Creative R2**: 별 시스템 + 히어로 해금.

**Orchestrator 판정**: v1 스코프 외로 확정.
**근거**: 핵심 루프 검증이 우선이다. 메타 진행도는 좋은 확장 방향이나 기본 게임이 재미있어야 의미가 있다.

## 최종 합의 설계 요약

3라운드에 걸친 토론 결과, **Classic의 핵심 원칙(Klondike 규칙 보존, 명확한 투사체 매핑, 10분 세션, 레벨업 시스템)** 위에 **Creative의 혁신 메카닉(양방향 피드백, 체인 게이지, 콤보 타이머, 적 약점 시스템, 결계 존)**을 레이어링하는 방향으로 합의되었다.

불채택된 Creative 요소(부정적 역방향, 6+ 콤보 조합, 12분 세션, 메타 진행도)는 "인지 부하 과잉", "하향 나선 위험", "구현 스코프 초과"라는 Classic의 비판이 타당했다.

채택된 Creative 요소(긍정적 역방향, 콤보 타이머, 적 약점, 체인 게이지)는 모두 "Klondike 규칙을 변경하지 않으면서 전투와의 연결을 강화"하는 것들로, Classic의 원칙과 양립 가능했다.

기존 디펜스 모드와 비교하여 가장 큰 차이점은: (1) 턴제→실시간 전환, (2) 유닛 소환→투사체 강화 전환, (3) 양방향 피드백 도입이다.
