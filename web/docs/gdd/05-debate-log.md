# 토론 과정 기록 (Appendix)

## 참여자

- **Classic Rule Agent**: 보수적 클래식 룰 수호자. 검증된 메카닉, 규칙 단순성, 원작 존중.
- **Creative Rule Agent**: 혁신적 기획자. 차별화, 장르 시너지, 리플레이 가치.
- **Orchestrator**: 합의 판정 및 문서화 관리.

---

## Round 1: 독립 제안

### Classic Agent 초안

**핵심 구조:**
- 표준 클론다이크 규칙 완전 유지
- Foundation 카드 올리면 SP +1, Tableau 뒤집으면 Mana +1
- Stock 완전 순환 시 웨이브 타이머 -5초 (패널티)
- "준비 완료" 버튼 턴제 진행
- 몬스터: 직선 경로, Foundation 영역 도달 시 HP -1 (HP 3)

**유닛 시스템:**
- 2-Suit 조합 (6가지) × 4티어 = 24종
- Foundation 최소 랭크 = 티어
- Mana로 소환 비용 지불
- 고정 슬롯 5-7개 배치

**스코어링:**
- 처치 수 × Foundation 완성도 보너스(×1.0~3.0) + 생존 보너스

**자가 진단:**
- "이게 정말 솔리테어인가?" — Suit 조합 의식하면 원작 왜곡
- 디펜스 파트가 형식적일 가능성
- Mana 시스템이 불필요한 복잡성
- 최종 의견: "디펜스 파트가 정말 필요한가?" 의문 제기

---

### Creative Agent 초안: "Arcane Sequence"

**핵심 구조:**
- 클래식 Klondike 기반, 실시간 진행 (몬스터가 다가오는 동안 카드 정리)
- Foundation 올리면 해당 Suit 마나 게이지 +1
- "카드 순서 = 유닛 빌드"

**유닛 시스템:**
- Suit별 역할: Hearts=회복, Diamonds=원거리, Clubs=근접, Spades=방어
- 2~3 Suit "동시 도달" → 합성 유닛
- 4 Suit 모두 7 도달 → "세븐즈 아포칼립스" (전체 제거, 게임당 1회)
- Foundation A→K 완성 = 궁극 유닛

**스코어링:**
- 처치 + 콤보 보너스 - 페널티(Core 피해, Undo)
- 시간 계수 (빠를수록 높음)

**차별화 포인트:**
- 카드 순서 = 유닛 빌드 (메타 전략)
- 동시 도달 합성 (긴장감)
- 시간 압박 솔리테어 (장르 전복)
- 카드 덱 = 유닛 풀 (로그라이크)
- Foundation 완성 = 궁극기

**자가 진단:**
- 동시 진행 인지 부하
- 솔리테어 운빨 vs 디펜스 실력 불균형
- 합성 타이밍 운 요소

## Round 2: 교차 비판

### Classic → Creative 비판

**인정한 점:**
1. "카드 순서 = 유닛 빌드" 핵심 개념 — 솔리테어 전략을 디펜스로 자연스럽게 확장
2. 세븐즈 아포칼립스 궁극기 — 4-Suit 병렬 관리를 유도하는 극적 보상

**지적한 문제점:**
1. "동시 도달" 합성 조건이 비결정론적 — 시간 기반 조건은 클론다이크 본질(결정론적 규칙)에 위배
2. 실시간 진행이 솔리테어 사고 시간을 침해 — "반사신경 게임"으로 변질

**제시한 대안:**
- "동시 도달" → "동등 레벨 보유" 상태 조건으로 변경
- 실시간 → 턴제 하이브리드 (솔리테어 시간정지 + 디펜스 자동진행)

**질문:** "Undo -10점 페널티로 허용하는 이유? 디펜스 상태까지 되돌려야 하므로 구현 복잡도 급증 아닌가?"

---

### Creative → Classic 비판

**인정한 점:**
1. 2-Suit 조합 시스템의 명확성 — 6가지 조합이 직관적이고 Foundation-디펜스 연결이 견고
2. "준비 완료" 버튼의 전략적 타이밍 — 솔리테어 집중 공간 확보

**지적한 문제점:**
1. Suit 조합 의식 시 솔리테어 재미 왜곡 — "조합 맞추기 게임"으로 전락
2. Mana 시스템의 불필요한 복잡성 — Tableau 뒤집기 = Mana는 운에 의존, 시너지 약함

**제시한 대안:**
- Foundation 완성 순서 자체를 보상으로 → Suit 조합은 사후적 평가
- Mana 제거 → Foundation 카드 자체를 Energy로 단순화 + Spell 토큰 신설

**질문:** "Stock 순환 패널티 -5초는 운이 나빠 막힌 플레이어에게 추가 불이익 아닌가?"

## Orchestrator 최종 판정

### 판정 1: 유닛 소환 방식 → Classic 방향 (2-Suit 조합)
- **근거**: 6가지 명시적 조합이 전략 분기를 명확하게 만듦. Creative의 "완성 순서로 자동 결정"은 우아하지만, 플레이어가 조합을 예측하기 어려워 전략적 깊이가 얕아짐. Classic 방식은 "♠+♥를 노릴까 ♣+♦를 노릴까?"라는 명확한 의사결정을 제공.
- **Creative 우려 반영**: 어떤 조합이든 유용하게 밸런싱하여 "정답 빌드" 고착 방지.

### 판정 2: 합성 조건 → Classic 수정안 (동등 레벨 보유)
- **근거**: "동시 도달" 타이밍은 비결정론적이라는 Classic 비판이 타당. "동등 레벨 보유 시 다음 카드 올리면 합성"은 결정론적이면서 계획 가능.

### 판정 3: Spell 토큰 → Creative 채택
- **근거**: Tableau 뒤집기에 디펜스 가치를 부여하는 핵심 양방향 시너지 장치. Mana 제거 후 빈 자리를 Spell이 채우며, Recall은 Undo 대체 안전장치로 기능.

### 판정 4: Undo → Classic 채택 (제거)
- **근거**: 턴제 구조로 사고 시간이 충분하고, Recall Spell이 Foundation 카드 복귀 기능을 제공. Undo 허용 시 디펜스 상태 롤백 구현 복잡도가 과도.

### 판정 5: Stock 순환 패널티 → Creative 채택 (제거)
- **근거**: Stock 순환은 막힌 상황에서 불가피한 선택. 플레이어 통제 밖의 행위에 패널티를 부과하면 좌절감만 유발.

### 판정 6: 스코어 시간 계수 → Creative 채택 (제거)
- **근거**: 턴제 구조에서 "빨리 풀기"를 보상하면 턴제의 핵심 가치(충분한 사고 시간)와 모순.

### 판정 7: 세븐즈 아포칼립스 → 포함
- **근거**: 4-Suit 병렬 관리를 유도하는 마일스톤 보상. 양측 R1에서 긍정 평가.

### 판정 8: 동일 조합 복수 소환 → Classic 채택 (허용, 효율 감소)
- **근거**: 1종만 허용하면 전략 유연성 제한. 복수 소환 허용하되 2번째부터 효율 70%로 감소시켜 다양한 조합 사용을 유도.

## 최종 합의 구조 요약

```
솔리테어 페이즈 (턴제, 시간 정지)
├── Foundation 카드 올리기 → Energy +1
├── Tableau 뒤집기 → Spell 토큰 +1
├── 2-Suit 동등 레벨 도달 → 합성 유닛 소환 가능
└── "준비 완료" → 디펜스 페이즈 전환

디펜스 페이즈 (자동 전투, 10~15초)
├── 유닛 자동 공격
├── Spell 토큰으로 전술 개입 (Freeze/Boost)
├── 몬스터 처치 → 스코어
└── Core HP 0 → 게임 오버

자원 경제
├── Energy: Foundation 카드 → 유닛 소환 비용
├── Spell 토큰: Tableau 뒤집기 → 전술 스펠
└── Foundation 스택: 소모 안 됨 (레벨 유지)
```
